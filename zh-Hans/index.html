OmniContext CLI 是一个上下文优先、零遥测的终端编程助手。它尽量保持轻量，原生支持 Anthropic、OpenAI、Gemini 和 Responses API， 也能在需要时延伸到 VS Code、Office、浏览器和移动端。

$ npm install -g omni-context-cli && omx

工作流

内置工作流先上手，
需要时再完全自定义

工作流本质上就是 markdown：frontmatter 控制工具权限，正文定义行为。 OmniContext CLI 默认内置 Programming、General、Analytics、Memory 和 Recall 五个工作流，桌面应用还会额外安装给 Chrome 侧边栏使用的 Browser 工作流，以及用于提醒和通知的 Memo 工作流。

○

Programming 默认

面向终端和编辑器场景的主力编程工作流。输出简洁、偏工具驱动，并且会读取 OMX.md 和 CLAUDE.md 这样的项目指令。

基础工具 智能体工具 自定义 Agent MCP 远程工具
◇

General

更通用的个人助理工作流，适合文档、表格、演示文稿和日常任务。风格清晰、友好，也会主动使用工具。

基础工具 Recall Agent 自定义 Agent MCP 远程工具
◆

Analytics

用量分析工作流，可直接查询本地请求日志数据库。适合了解 token 开销、模型使用情况、缓存命中率和延迟趋势。

AnalyticsQuery WebSearch Recall Agent
◇

Memory

用于管理项目记忆的工作流，适合定期查看、编辑和清理已经保存下来的关键要点，避免记忆逐渐变成噪音。

MemoryList MemoryUpdate MemoryDelete WebSearch
◈

Recall

聊天历史搜索工作流。通过全文检索在过去的会话中按关键词搜索，并提取相关的对话上下文。

RecallSearch RecallRead WebSearch
⬡

Browser

由桌面应用安装，专门给 Chrome 侧边栏使用。适合标签页控制、页面提取、书签、历史记录、截图，以及受限的页面脚本执行。

标签页 页面内容 书签 历史记录 截图

Memo

由桌面应用安装，用于个人提醒和通知。在 macOS 上，备忘事项会通过原生桥接直接同步进系统的"提醒事项"。

提醒 循环周期 优先级 系统通知 Apple 提醒事项

工具

直接动手的工具，
和替你做探索的智能体

基础工具负责直接执行动作。智能体工具则把探索性工作放到辅助模型上完成， 中间的工具调用、文件读取和草稿推理不会挤占你的主上下文。

智能体工具

Agent_Explore

梳理项目结构，并解释整个代码库是怎么组织起来的。

Agent_Glance

快速预览多个文件或目录，并输出简短摘要。

Agent_Search

围绕某个技术问题，在整个代码库里搜索答案。

Agent_Recall

搜索过去的聊天会话，并围绕某个主题总结发生了什么。

Agent_Review

用新的视角复查当前对话里最近完成的工作。

Agent_Deduce

通过只读探索追查 bug 或复杂问题，并给出修复建议。

内置工具

AnalyticsQuery

查询本地请求日志数据库，用于分析用量、延迟、缓存和成本。

Bash

运行 shell 命令，用于构建、测试、安装和各种终端任务。

BashOutput

查看后台 Bash 任务的输出，不会阻塞当前会话。

Edit

对现有文件做精确的文本替换。

Glob

按模式查找文件，并自动遵循 `.gitignore`。

Grep

用正则搜索文件内容，并返回路径和行号。

LandmarkList

列出已保存的地标 layer，方便快速定位代码结构。

LandmarkRead

读取某个地标 layer，包括它保存的文件、目录和符号。

LandmarkUpdate

创建或更新地标 layer，让稳定的导航知识可以跨会话保留。

LandmarkDelete

删除已经过时的地标 layer 和它保存的 pins。

MemoryList

列出当前项目里已保存的跨会话记忆要点。

MemoryUpdate

修改已有记忆要点的文本或分数。

MemoryDelete

删除过时或噪音过大的项目记忆要点。

Read

按行号读取文件，也支持 offset、limit、图片和 PDF。

RecallSearch

用全文检索搜索已经索引的历史会话。

RecallRead

读取某个 recall 搜索命中的过去会话上下文。

WebSearch

搜索最新的网页信息和在线文档。

WebFetch

抓取 URL，并把页面转换成干净的 markdown。

Write

从头创建文件或整文件覆盖，也支持更安全的 create-only 模式。

Skill

加载可复用的技能指令，也兼容 Claude Code 的技能格式。

上下文经济学

围绕上下文窗口而设计

OmniContext CLI 会从多个角度压缩上下文、控制成本：更短的提示词、智能体分流、自动压缩、可复用记忆，以及在合适的时候启用提示词缓存。

精简系统提示词

内置提示词尽量短、尽量聚焦，把 token 留给真正的问题，而不是模板噪音。

智能体工具

探索性工作会放到辅助模型上做，回到主对话里的只有最后结果。

上下文编辑

旧的工具负载和推理痕迹会被替换成紧凑占位符，保持上下文干净。

跨会话记忆

有价值的项目知识会自动带到下一次会话里，减少手工重复解释。

提示词缓存

你可以选择 5 分钟或 1 小时缓存窗口，让重复上下文更省钱。

跨会话记忆

把真正有用的内容留住

记忆会在压缩时被提取出来，随着时间打分，并在下一次会话里自动重新注入。 你也可以随时打开 Memory 工作流，手动查看、编辑或删除已经保存的要点。

自动提取 从过去的对话里提炼出真正有用的项目知识，自动带到以后。
分数机制 有帮助的记忆 +3，中性的 -1，有害的 -6，到 -10 就会被清除。
需要时可手动清理 Memory 工作流会直接提供 MemoryList、MemoryUpdate 和 MemoryDelete。
按项目存储 每个项目都会把记忆存放在 ~/.omx/projects/<project-id>/memory.json 里。
memory.json +3
"This repo uses OMX.md for project instructions and falls back to CLAUDE.md."
memory.json +2
"The backend uses SQLite stores under ~/.omx/projects/<project-id>/ for project state."
memory.json -8
"This project deploys through Webpack." 衰减中 — 到 -10 会被清除
地标导航

保存地图，
不只保存答案

Landmarks 会把反复探索出来的代码结构整理成可复用的导航 layer。以后再遇到同一个子系统时， OmniContext CLI 可以直接回到那组关键文件、目录和符号，而不是再从头摸一遍。

按概念建 layer 把某个功能、子系统或请求链路保存成一个独立 layer，范围和用途都清楚可读。
精确 pin 到代码对象 每个 layer 都可以 pin 到文件、目录和符号，导航会比普通笔记具体得多。
完整工具链 直接用 LandmarkList、LandmarkRead、LandmarkUpdate 和 LandmarkDelete 管理它。
按项目 SQLite 存储 每个项目都会把自己的导航地图保存在 ~/.omx/projects/<project-id>/landmark.sqlite。
Landmark workflow
# 先看当前已经有哪些地图
$ LandmarkList
Navigation Request Lifecycle - Traces how requests move through the app

# 读出其中一个 layer
$ LandmarkRead Navigation Request Lifecycle
[symbol] source/services/chatOrchestrator.ts > ChatOrchestrator - Main request flow

# 真的摸清结构后，再把地图存下来
$ LandmarkUpdate Navigation Request Lifecycle ...
集成

一个助手，
覆盖你原本的工作流

终端依然是重心，但 OmniContext CLI 也能延伸到编辑器、浏览器、Office 和移动端。 桌面应用会把这些能力串起来，并启动它们共享的本地服务。

桌面应用

工作区管理、模型配置、审批模式、局域网访问、备忘与提醒系统，以及为整套集成一键启动共享服务。

工作区 模型配置 审批模式 局域网访问 备忘与提醒 服务启动

VS Code 扩展

把文件上下文、选区、诊断信息和 diff 视图直接送进对话里。

活动文件 选区 诊断信息 Diff 视图

Chrome 扩展

在任意页面上使用侧边栏，支持标签页控制、页面提取、书签、历史记录、截图以及受限的页面脚本执行。

Office 加载项

通过桌面应用和共享本地服务，把 OmniContext CLI 带进 Word、Excel 和 PowerPoint。

Web 客户端

在浏览器里使用 OmniContext CLI，支持附件、Mermaid、LaTeX、markdown 渲染和页头自定义状态栏。

移动端访问

运行 omx --serve --host 0.0.0.0 后，就能在本地网络或通过 Tailscale 从手机和平板接入。

可扩展性

一边写 markdown，
一边长能力

工作流、自定义 Agent、技能和 MCP 服务器都是开放的。 OmniContext CLI 的设计目标之一，就是让你在不碰核心代码的情况下完成扩展。

自定义工作流

定义你自己的系统提示词，并且精确白名单化可用工具。

~/.omx/workflows/my-flow.md

自定义智能体工具

写一个 markdown 提示模板，再加 schema 和工具权限，就能像内置 Agent 一样调用。

~/.omx/agents/review.md

技能

把可复用的提示指令做成斜杠命令，或者通过 Skill 工具加载。Claude Code 技能也能直接用。

~/.omx/skills/review/SKILL.md

MCP 服务器

通过 stdio 或 HTTP 接外部工具和数据源，也会自动读取 Claude 的 MCP 配置。

~/.omx/mcp.json
细节体验

真正好用，往往在这些小地方

精简系统提示词

更短的提示词、更少的模板负担，把上下文空间留给真正的问题。

零遥测

没有分析埋点、没有使用追踪，也没有额外套在工作流上的遥测层。

上下文编辑

裁掉旧的工具负载和推理痕迹，让当前上下文一直保持清爽。

自动压缩

当上下文变拥挤时，OmniContext CLI 会自动总结会话，并把关键内容带到下一轮。

扩展思考

当前模型支持时，可以为更复杂的问题开启更深一层的推理模式。

兼容 CLAUDE.md

如果仓库里已经有 CLAUDE.md，OmniContext CLI 会直接读取；加上 OMX.md 后则优先使用它。

提示词缓存

你可以在 5 分钟和 1 小时缓存窗口之间切换，减少重复提示的成本。

颜色主题

内置 crystal、splash、neon、forest 等配色，终端和 Web 客户端都能统一使用。
