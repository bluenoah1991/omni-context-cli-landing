OmniContext CLI 是一个上下文优先、零遥测的终端编程助手。它尽量保持轻量，原生支持 Anthropic、OpenAI、Gemini 和 Responses API， 也能在需要时延伸到 VS Code、Office、浏览器和移动端。

$ npm install -g omni-context-cli && omx

工作流

内置工作流先上手，
需要时再完全自定义

工作流本质上就是 markdown：frontmatter 控制工具权限，正文定义行为。 OmniContext CLI 默认内置 Programming 和 General，桌面应用还会额外安装一个给 Chrome 侧边栏使用的 Browser 工作流。

○

Programming 默认

面向终端和编辑器场景的主力编程工作流。输出简洁、偏工具驱动，并且会读取 OMX.md 和 CLAUDE.md 这样的项目指令。

基础工具 智能体工具 自定义 Agent MCP 远程工具
◇

General

更通用的个人助理工作流，适合文档、表格、演示文稿和日常任务。风格清晰、友好，也会主动使用工具。

基础工具 自定义 Agent MCP 远程工具
⬡

Browser

由桌面应用安装，专门给 Chrome 侧边栏使用。适合标签页控制、页面提取、书签、历史记录和截图。

标签页 页面内容 书签 历史记录 截图
+

Custom

把文件放进 ~/.omx/workflows/ 或 .omx/workflows/，它就会出现在工作流列表里。文件名本身就是你传给 --workflow 的工作流 ID。

name: Review
allowBaseTools: [Read, Glob, Grep, Skill]

工具

直接动手的工具，
和替你做探索的智能体

基础工具负责直接执行动作。智能体工具则把探索性工作放到辅助模型上完成， 中间的工具调用、文件读取和草稿推理不会挤占你的主上下文。

智能体工具

Agent_Explore

梳理项目结构，并解释整个代码库是怎么组织起来的。

Agent_Glance

快速预览多个文件或目录，并输出简短摘要。

Agent_Search

围绕某个技术问题，在整个代码库里搜索答案。

基础工具

Bash

运行 shell 命令，用于构建、测试、安装和各种终端任务。

BashOutput

查看后台 Bash 任务的输出，不会阻塞当前会话。

Read

按行号读取文件，也支持 offset、limit、图片和 PDF。

Edit

对现有文件做精确的文本替换。

Write

从头创建文件或整文件覆盖，也支持更安全的 create-only 模式。

Glob

按模式查找文件，并自动遵循 `.gitignore`。

Grep

用正则搜索文件内容，并返回路径和行号。

WebSearch

搜索最新的网页信息和在线文档。

WebFetch

抓取 URL，并把页面转换成干净的 markdown。

Skill

加载可复用的技能指令，也兼容 Claude Code 的技能格式。

原生多协议

四种 API 协议，
没有中间翻译层

每一种协议都有自己的请求构建器和流处理器，所以提示词缓存、扩展思考、结构化输出、工具编排这些行为， 都会按供应商原本设计的方式工作。

Anthropic
Messages API，支持提示词缓存、扩展思考和流式输出。
Claude Opus / Sonnet / Haiku
OpenAI
Chat Completions，兼容各种 OpenAI 风格端点，包括本地和托管供应商。
GPT-4o / GLM / DeepSeek / MiniMax
Gemini
原生 generateContent 支持，保留 Gemini 自己的流式和函数调用风格。
Gemini Pro / Flash
Responses API
OpenAI 的 Responses API，使用独立请求路径和原生工具编排能力。
GPT-4.1 / GPT-4o / Codex
上下文经济学

围绕上下文窗口而设计

OmniContext CLI 会从多个角度压缩上下文、控制成本：更短的提示词、智能体分流、自动压缩、可复用记忆，以及在合适的时候启用提示词缓存。

精简系统提示词

内置提示词尽量短、尽量聚焦，把 token 留给真正的问题，而不是模板噪音。

智能体工具

探索性工作会放到辅助模型上做，回到主对话里的只有最后结果。

上下文编辑

旧的工具负载和推理痕迹会被替换成紧凑占位符，保持上下文干净。

跨会话记忆

有价值的项目知识会自动带到下一次会话里，减少手工重复解释。

提示词缓存

你可以选择 5 分钟或 1 小时缓存窗口，让重复上下文更省钱。

模型供应商

导入整组模型

内置供应商预设可以让第一次配置更快。贴一个 API key，OmniContext CLI 就能把对应模型批量导入进来。

快速开始
# 查看内置供应商
$ omx --list-providers

# 一次性导入某个供应商的全部模型
$ omx --add-provider zenmux --api-key zmx-...
Added 20+ models from Zenmux

# 不想要了也可以整组移除
$ omx --remove-provider zenmux
跨会话记忆

把真正有用的内容留住

记忆会在压缩时被提取出来，随着时间打分，并在下一次会话里自动重新注入。 有帮助的内容会升值，陈旧内容会衰减，错误建议会更快被清掉。

自动提取 从过去的对话里提炼出真正有用的项目知识，自动带到以后。
分数机制 有帮助的记忆 +3，中性的 -1，有害的 -6。
按项目存储 每个项目都有自己的 .omx/memory.json 文件。
memory.json +3
"This repo uses OMX.md for project instructions and falls back to CLAUDE.md."
memory.json +2
"The web client footer shows statusCommand output from omx.json."
memory.json -8
"This project deploys through Webpack." 衰减中 — 到 -10 会被清除
集成

一个助手，
覆盖你原本的工作流

终端依然是重心，但 OmniContext CLI 也能延伸到编辑器、浏览器、Office 和移动端。 桌面应用会把这些能力串起来，并启动它们共享的本地服务。

桌面应用

工作区管理、模型配置、备忘和提醒工具，以及一键启动共享服务。

工作区 模型配置 备忘录 服务启动

VS Code 扩展

把文件上下文、选区、诊断信息和 diff 视图直接送进对话里。

活动文件 选区 诊断信息 Diff 视图

Chrome 扩展

在任意页面上使用侧边栏，支持标签页控制、页面提取、书签、历史记录和截图。

Office 加载项

通过桌面应用把 OmniContext CLI 带进 Word、Excel 和 PowerPoint。

Zed 编辑器

把 OmniContext CLI 作为 ACP agent 跑在 Zed 里，工具访问能力和终端保持一致。

Web 客户端

在浏览器里使用 OmniContext CLI，支持附件、 Mermaid、LaTeX、markdown 渲染和自定义状态栏。

移动端访问

运行 omx --serve 后，就能在本地网络或通过 Tailscale 从手机和平板接入。

可扩展性

一边写 markdown，
一边长能力

工作流、自定义 Agent、技能和 MCP 服务器都是开放的。 OmniContext CLI 的设计目标之一，就是让你在不碰核心代码的情况下完成扩展。

自定义工作流

定义你自己的系统提示词，并且精确白名单化可用工具。

~/.OmniContext CLI/workflows/my-flow.md

自定义智能体工具

写一个 markdown 提示模板，再加 schema 和工具权限，就能像内置 Agent 一样调用。

~/.OmniContext CLI/agents/review.md

技能

把可复用的提示指令做成斜杠命令，或者通过 Skill 工具加载。Claude Code 技能也能直接用。

~/.OmniContext CLI/skills/review/SKILL.md

MCP 服务器

通过 stdio 或 HTTP 接外部工具和数据源，也会自动读取 Claude 的 MCP 配置。

~/.OmniContext CLI/mcp.json
细节体验

真正好用，往往在这些小地方

精简系统提示词

更短的提示词、更少的模板负担，把上下文空间留给真正的问题。

零遥测

没有分析埋点、没有使用追踪，也没有额外套在工作流上的遥测层。

上下文编辑

裁掉旧的工具负载和推理痕迹，让当前上下文一直保持清爽。

自动压缩

当上下文变拥挤时，OmniContext CLI 会自动总结会话，并把关键内容带到下一轮。

扩展思考

当前模型支持时，可以为更复杂的问题开启更深一层的推理模式。

兼容 CLAUDE.md

如果仓库里已经有 CLAUDE.md，OmniContext CLI 会直接读取；加上 OMX.md 后则优先使用它。

提示词缓存

你可以在 5 分钟和 1 小时缓存窗口之间切换，减少重复提示的成本。

颜色主题

内置 crystal、splash、neon、forest 等配色，终端和 Web 客户端都能统一使用。
