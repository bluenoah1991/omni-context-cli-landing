工具

基础工具 + 智能体工具

基础工具提供文件系统和网络访问。智能体式工具在更便宜的模型上运行探索性任务—— 中间工具调用、文件内容和推理过程不会进入主上下文。 你只拿到一个简洁的结论，中间草稿完全不占空间。

智能体工具

Agent_Explore

概览项目架构。映射目录布局、关键文件和代码库组织方式。

Agent_Glance

一次预览多个文件并生成简要摘要。快速了解你正在处理的内容。

Agent_Search

搜索代码库以回答特定问题。从项目各处返回有针对性的代码片段。

基础工具

Bash

运行 Shell 命令。适用于构建、安装、CLI 工具和系统任务。

Read

读取文件内容并显示行号。支持偏移和限制参数以浏览大文件。

Edit

精确替换文本。提供要查找的确切文本和替换内容。

Write

将内容写入文件，自动创建父目录。适合新建文件和完整重写。

Glob

按模式查找文件。标准 glob 语法，支持 gitignore。

Grep

使用正则表达式搜索文件内容。返回匹配项及文件路径和行号。

WebSearch

搜索网络以获取最新信息和文档。

WebFetch

获取 URL 内容并将 HTML 转换为干净的 Markdown。

Skill

加载技能以获取特定领域任务的详细指导。

原生多协议

四种 API 协议，
零格式转换

每种 LLM API 协议都有专用的请求构建器和流处理器。 提示词缓存、扩展思考、结构化输出和每一个供应商专属特性都按原厂设计工作。 不会把 Anthropic 请求转成 OpenAI 格式，也不会反过来。

Anthropic
Messages API，支持提示缓存、扩展思考和流式输出。 Token 级别的缓存控制，自定义 TTL。
Claude Opus / Sonnet / Haiku
OpenAI
Chat Completions API。兼容任何 OpenAI 格式的端点—— 本地模型、智谱、MiniMax 等。
GPT-4o / GLM / MiniMax / DeepSeek
Gemini
原生 generateContent API，Gemini 专属流式处理。 工具调用使用 Gemini 自己的 Schema，不用 OpenAI 适配层。
Gemini Pro / Flash
Responses API
OpenAI 的新一代 Responses API，内置工具编排。 独立于 Chat Completions 的路径，不是兼容包装器。
GPT-4o / Codex
成本优化

上下文优先设计，真正省钱

每次 API 调用都会重新发送完整对话历史。OmniContext CLI 从四个维度控制成本——保持上下文精简、减少轮次、压低账单。

精简系统提示词

内置提示词短小、聚焦、没有模板废话。 工具描述力求精简。你的上下文预算用在实际工作上， 而不是框架开销。

智能体式工具

探索性任务由智能体式工具在更便宜的模型上运行。 你只拿到一个简洁的结论，中间草稿完全不占空间。 贵价模型永远不做苦力活。

上下文编辑

随着对话增长，早期的工具调用负载和思考块变得不再相关。 上下文编辑用紧凑的占位符替换庞大的工具输入输出， 让模型看到干净、聚焦的历史。

跨会话记忆

好的见解不断积累权重，过时的知识自然衰减。 积累的记忆自动注入，让助手从第一句话就了解 你的偏好和项目背景。

模型供应商

一条命令
添加所有模型

OmniContext CLI 内置供应商预设。选择一个，粘贴 API Key， 该服务的所有模型就可以用了。

Zenmux
DeepSeek
Kimi for Coding
OpenRouter
智谱 (GLM)

快速设置
# 列出可用供应商
$ omx --list-providers

# 一键添加供应商所有模型
$ omx --add-provider zenmux --api-key zmx-...
Added: Zenmux Anthropic (Claude Sonnet 4)
Added: Zenmux Gemini (Gemini 2.5 Flash)
...

# 移除供应商
$ omx --remove-provider zenmux
跨会话记忆

边用边学

记忆提取在对话压缩时自动触发。模型回顾当前会话，提炼关键要点， 并评估已有的要点。每个要点的评分根据它在对话中的表现变动。

自动提取 每次压缩时自动提取关键点，并注入到后续会话中
评分记忆 有用的 +3，中性的 -1，有害的 -6。 低于 -10 自动清除。
按项目存储 每个项目有独立的记忆文件 .omx/memory.json。 直接编辑可完全控制。
memory.json +3
"This project uses TypeScript strict mode with path aliases configured in tsconfig"
memory.json +2
"API routes follow REST conventions in src/routes/ with Zod validation"
memory.json -8
"Uses Webpack for bundling" 衰减中——低于 -10 将被清除

可扩展性

自定义工作流、智能体、技能和 MCP 服务器。 在 OmniContext CLI 之上构建，无需触碰核心。

自定义工作流

定义自己的系统提示词，控制哪些工具可用。 YAML 前置信息 + Markdown 正文。

~/.omx/workflows/my-flow.md

智能体式工具

写一个带提示词模板、参数定义和工具权限的 Markdown 文件， 立刻成为可调用的智能体式工具。使用 AGENTS.md 提供 Agent 指令，缺失时回退到 OMX.md 和 CLAUDE.md。

~/.omx/agents/review.md

技能 (Skills)

基于目录的提示能力，通过斜杠命令调用。 注入特定领域的知识指导。兼容 Claude Code 技能格式。

~/.omx/skills/style/SKILL.md

MCP 服务器

通过 Model Context Protocol 连接外部工具和数据源。 支持 Stdio 和 HTTP 传输。也读取 Claude 的 MCP 配置。

~/.omx/mcp.json

兼容 CLAUDE.md

已有 CLAUDE.md？OmniContext CLI 会自动读取，与 OMX.md 并行使用。零迁移成本。

提示词缓存

你可以自主选择缓存 TTL：5 分钟适合快速迭代， 1 小时适合长时间工作。重复的上下文块直接从缓存读取。

颜色主题

17 种内置颜色预设，适用于终端和 Web UI。 Crystal、Neon、Sunset 等。支持亮色和暗色模式。
