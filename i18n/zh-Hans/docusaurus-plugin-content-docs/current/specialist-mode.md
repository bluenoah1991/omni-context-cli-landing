# 专家模式

专家模式是 Omx 的核心功能。它用智能体工具替代基本的文件操作，这些工具能够理解上下文、处理错误并自主完成任务。

## 工作原理

在传统的编程助手中，模型调用低级工具如 `read_file`、`write_file` 和 `bash`。每次工具调用都会消耗上下文 token，并且需要模型手动处理错误。

专家模式引入了 **智能体工具（Agentic Tools）** - 封装复杂工作流的高级工具。当你向 Omx 提出请求时，它会委托给专门的工具：

1. 理解你的意图
2. 执行必要的操作
3. 自动处理错误和重试
4. 返回干净的结果

## 可用的智能体工具

| 工具 | 用途 |
|-------|---------:|
| **craft** | 执行 bash 命令，失败时自动重试 |
| **sculpt** | 编辑文件，自动纠错 |
| **weave** | 写入文件，具有自动修复功能 |
| **sweep** | 查找匹配语义条件的文件 |
| **pluck** | 提取特定代码片段 |
| **explore** | 探索项目结构和架构 |
| **ripple** | 查找符号的所有引用 |
| **unravel** | 追踪函数调用依赖 |
| **quest** | 通过网络搜索研究主题 |
| **graphic** | 生成 Mermaid 图表 |

## 启用专家模式

专家模式默认启用。你可以使用 `/status` 查看当前设置，或通过菜单切换（按 `Escape` 打开菜单）。

禁用时，Omx 直接使用基本文件工具，给你更精细的控制，但需要更多上下文 token。
