# 专家模式

专家模式是 Omx 的核心功能。它用智能体工具替代基本的文件操作，减少多轮来回。

## 工作原理

在传统的编程助手中，模型调用低级工具如 `read`、`edit`、`write` 和 `bash`。每次工具调用都会消耗上下文 token，并且需要模型手动处理错误。

专家模式引入了 **智能体工具**，用高级工具封装复杂工作流。当你向 Omx 提出请求时，它会委托给专门的工具：

1. 理解你的意图
2. 执行必要的操作
3. 自动处理错误和重试
4. 返回干净的结果

## 可用的智能体工具

| 工具 | 用途 |
|-------|---------:|
| **spark** | 执行 bash 命令，失败时自动重试 |
| **sculpt** | 替换文本编辑文件，匹配失败会自动修复 |
| **weave** | 写入文件，失败会重试 |
| **sweep** | 按条件查找文件 |
| **pluck** | 从文件中提取代码片段 |
| **explore** | 探索项目结构和架构 |
| **ripple** | 查找符号的所有引用 |
| **slice** | 提取与回答特定问题相关的代码片段 |
| **glance** | 预览多个文件并给出摘要 |
| **quest** | 通过网络搜索研究主题 |

## 启用专家模式

专家模式默认启用。你可以使用 `/status` 查看当前设置，或通过菜单切换预设（按 `Escape` 打开菜单）。

禁用时，Omx 直接使用基本文件工具，给你更精细的控制，但需要更多上下文 token。
