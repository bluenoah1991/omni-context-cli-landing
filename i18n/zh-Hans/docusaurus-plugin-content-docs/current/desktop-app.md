# 桌面应用

桌面应用是把 OmniContext CLI 作为共享本地服务跑起来的最省事方式，也是整套生态里最自然的控制面板。

相比自己分别配置浏览器、Office 和移动端访问，桌面应用可以帮你把服务启动和各类集成串起来。

## 它负责什么

桌面应用本质上是 OmniContext CLI 的一个轻量启动器和控制台。它可以：

- 启动和停止本地 OmniContext CLI 服务
- 管理工作区
- 辅助模型配置
- 切换审批模式
- 为手机和平板开启局域网访问
- 打开内置浏览器扩展目录
- 安装和卸载 Office 加载项
- 自动安装 Browser 和 Memo 工作流
- 提供内置的备忘与提醒系统，并通过桌面通知提示
- 把浏览器、Office 等远程工具桥接到同一个 OmniContext CLI 实例

## 什么时候适合用

如果你平时主要在终端里工作，桌面应用不一定是每天都必须开的。

但当你希望 OmniContext CLI 变成一个跨多个入口共享的助手时，它会非常方便，比如：

- 终端
- 浏览器侧边栏
- Office 面板
- 移动端 Web 客户端

这样模型、会话和记忆都会挂在同一个运行中的实例上。

## 浏览器和 Office 的推荐入口

桌面应用是这些集成最推荐的入口：

- **Chrome 扩展** - 直接从桌面应用打开扩展目录，再去 Chrome 里加载
- **Office 加载项** - 在桌面应用的集成区域里完成安装或卸载

尤其是 Office，桌面应用会顺手帮你处理本地 HTTPS 和证书配置。

## Browser 工作流

桌面应用启动时会把 **Browser** 工作流安装到 `~/.omx/workflows/`。

这个工作流专门为页面阅读、标签页操作、书签、历史记录、截图和浏览器远程工具做过调优，所以在 Chrome 侧边栏里通常就是最合适的默认选择。

## Memo 工作流

桌面应用还会顺手安装一个 **Memo** 工作流，并通过 `Desktop_AddMemo`、`Desktop_ListMemos`、`Desktop_DeleteMemo`、`Desktop_Notify` 等工具暴露能力。

你可以用它快速添加备忘、设置带优先级（low、medium、high）的提醒，以及创建循环提醒（每天、每周、每两周、每月、每三个月、每半年或每年）。提醒到期时会以原生桌面通知的形式弹出。

在 macOS 上，备忘会通过一个原生 Swift 桥接直接写入系统的 **提醒事项**，因此可以和系统提醒事项 App、iPhone、iPad 自动同步。其他平台则会回退到 `~/.omx/` 下的本地 JSON 文件。

第一次在 macOS 上添加提醒时，系统会请求"提醒事项"的访问权限。授权一次之后，桌面应用就会自动接管后续的同步流程。

## 移动端访问

桌面应用可以把本地服务暴露到局域网，这样你就能更方便地在手机或平板上打开 OmniContext CLI，而不需要手动敲命令。

如果你需要跨网络但更安全的访问方式，建议再配合 Tailscale。

## 建议

- 想要一键控制服务时，用桌面应用
- 想要最轻量的工作流时，直接用终端
- 主要做文档、表格或演示时，优先用 **General** 工作流
- 在浏览器侧边栏里时，切到 **Browser**
- 想管理个人提醒和待办事项时，切到 **Memo**
