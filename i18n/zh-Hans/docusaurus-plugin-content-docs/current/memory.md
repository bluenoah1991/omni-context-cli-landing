# 跨会话记忆

OmniContext CLI 可以在多个会话之间记住有用的项目知识。记忆会在对话压缩时被提取出来，并在后续会话里自动重新注入。

记忆默认就是开启的。如果你不想在某个项目里使用它，也可以在偏好设置里关掉。

## 工作原理

每条记忆都有一个分数，会根据它是否真正有帮助而变化：

| 评价 | 分数变化 |
|------|---------|
| 有帮助 | +3 |
| 中性 | -1 |
| 有害 | -6 |

分数到 `-10` 或更低的记忆会在反思阶段被清除。这样有价值的记忆会随着时间持续积累，而过时或错误的指引则会自然衰减。

每条记忆都包含名字、文本和分数。其中文本会原样注入到未来会话的提示词里，所以保持记忆干净很重要。

## 如何管理记忆

最直接的方式是使用内置的 **Memory** 工作流：

```bash
omx --workflow memory
```

这个工作流主要围绕三个工具展开：

- `MemoryList` - 查看已有要点
- `MemoryUpdate` - 修改已有要点的文本或分数
- `MemoryDelete` - 删除过时或噪音太大的要点

当然，你也可以完全不手动管理。只要记忆功能保持开启，OmniContext CLI 就会在压缩时自动提取和清理这些要点。

## 如何开启或关闭

你可以在偏好设置菜单里切换记忆功能：

1. 按 `Escape`
2. 选择 **Change preferences**
3. 切换 **Cross-session memory**

## 存储位置

每个项目都会把自己的记忆存放在 `~/.omx/projects/<project-id>/memory.json` 中。

这个文件保存的是带版本号的 key points，每条都有名字、文本和分数。如果你想完全掌控，也可以直接手动编辑它。

## 会记住什么

模型会在压缩时自行判断哪些内容值得保留。常见例子包括：

- 项目结构和约定
- 技术选型和配置细节
- 反复出现的代码库特有模式
- 下次最好避开的失败尝试
- 之后仍然重要的用户偏好
