# 跨会话记忆

OmniContext CLI 可以在多个会话之间记住有用的项目知识。记忆会在对话压缩时被提取出来，并在后续会话里自动重新注入。

## 工作原理

每条记忆都有一个分数，会根据它是否真正有帮助而变化：

| 评价 | 分数变化 |
|------|---------|
| 有帮助 | +3 |
| 中性 | -1 |
| 有害 | -6 |

分数低于 `-10` 的记忆会被清除。这样有价值的记忆会随着时间持续积累，而过时或错误的指引则会自然衰减。

## 如何开启记忆

你可以在偏好设置菜单里切换记忆功能：

1. 按 `Escape`
2. 选择 **Change preferences**
3. 切换 **Cross-session memory**

## 存储位置

每个项目都会把自己的记忆存放在 `.omx/memory.json` 中。

这个文件是一个 JSON 数组，每一项都包含内容字符串和分数。如果你想完全掌控，也可以直接手动编辑它。

## 会记住什么

模型会在压缩时自行判断哪些内容值得保留。常见例子包括：

- 项目结构和约定
- 技术选型和配置
- 代码库特有的模式
- 重复出现的错误以及修正方式
- 对后续会话依然重要的用户偏好
