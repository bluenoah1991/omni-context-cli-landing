# 会话管理

Omx 自动保存你的对话，并提供导航和恢复会话的工具。

## 自动保存

每次对话都会自动保存。会话按项目组织，基于当前工作目录。

## 加载之前的会话

使用 `/session` 浏览和加载之前的会话：

```
/session
```

从列表中选择一个会话来恢复对话历史。

## 回退

使用 `/rewind` 回到当前会话的之前某个点：

```
/rewind
```

这会显示你的消息列表。选择一条会从该点创建新会话，保留所选之前的所有内容。

在以下情况使用回退：
- 对话走向了错误的方向
- 你想尝试不同的方法
- 发生错误，你想重试

## 压缩

当上下文窗口填满时，Omx 可以压缩对话历史。

### 自动压缩

当 token 使用超过模型上下文大小的 80% 时，Omx 会压缩对话，并以摘要开启新会话。

### 手动压缩

使用 `/compact` 手动触发压缩：

```
/compact
```

这在以下情况有用：
- 你想在复杂任务前减少上下文
- 对话积累了不相关的内容
- 你接近上下文限制

### 压缩配置

Omx 支持上下文编辑来减少 token 用量。启用后，每次请求都会自动裁剪旧消息中的工具调用和思考块。可以通过 `contextEditing` 设置来切换。

## 重新开始

使用 `/clear` 开始全新的会话：

```
/clear
```

这会清除终端并创建新的空会话。之前的会话会保存，可以用 `/session` 恢复。
