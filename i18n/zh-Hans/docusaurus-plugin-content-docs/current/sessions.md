# 会话管理

Omx 自动保存你的对话，并提供导航和恢复会话的工具。

## 自动保存

每次对话都会自动保存。会话按项目组织，基于当前工作目录或 git 仓库根目录。

## 加载之前的会话

使用 `/session` 浏览和加载之前的会话：

```
/session
```

从列表中选择一个会话来恢复对话历史。

## 回退

使用 `/rewind` 回到当前会话的之前某个点：

```
/rewind
```

这会显示你的消息列表。选择一条会从该点创建新会话，保留所选之前的所有内容。

在以下情况使用回退：
- 对话走向了错误的方向
- 你想尝试不同的方法
- 发生错误，你想重试

## 压缩

当上下文窗口填满时，Omx 可以压缩对话历史。

### 自动压缩

当 token 使用超过模型上下文大小的 80% 时，Omx 会提示进行压缩。这会总结对话同时保留关键上下文。

### 手动压缩

使用 `/compact` 手动触发压缩：

```
/compact
```

这在以下情况有用：
- 你想在复杂任务前减少上下文
- 对话积累了不相关的内容
- 你接近上下文限制

### 压缩配置

Omx 支持上下文编辑功能来优化压缩效果。可以通过 `contextEditing` 设置来启用或禁用此功能。

默认情况下，Omx 在压缩对话时会保留所有工具调用结果。如果你想在压缩时移除工具结果以减少 token 使用，可以在设置文件中手动调整相关参数。

## 重新开始

使用 `/clear` 开始全新的会话：

```
/clear
```

这会清除终端并创建新的空会话。之前的会话会保存，可以用 `/session` 恢复。
