Workflows

fully under your control

Cross-Session Memory

It keeps the good stuff

Memory is extracted during compaction, scored over time, and re-injected automatically on the next session. You can also open the Memory workflow to review, edit, or delete saved key points when they need cleanup.

Automatic extraction Useful project facts are pulled from past conversations and carried forward.
Scored memory Helpful memories gain +3, neutral ones drift at -1, harmful ones drop at -6, and points at -10 are pruned.
Manual cleanup when needed The Memory workflow gives you MemoryList, MemoryUpdate, and MemoryDelete for direct control.
Per-project storage Every project keeps its own memory file in ~/.omx/projects/<project-id>/memory.json.
memory.json +3
"This repo uses OMX.md for project instructions and falls back to CLAUDE.md."
memory.json +2
"The backend uses SQLite stores under ~/.omx/projects/<project-id>/ for project state."
memory.json -8
"This project deploys through Webpack." Decaying — pruned at -10
Landmark Navigation

Save the map,
not just the answer

Landmarks turn repeat codebase discoveries into reusable navigation layers. Instead of relearning where a subsystem lives, OmniContext CLI can save a named layer with the key files, directories, and symbols, then jump back to it in later sessions.

Named layers Group a feature, subsystem, or request flow into a reusable layer with its own scope and description.
Pinned code objects Each layer stores pins for files, directories, and symbols, so navigation stays concrete instead of vague.
First-class tool flow Use LandmarkList, LandmarkRead, LandmarkUpdate, and LandmarkDelete directly from the coding workflow.
Per-project SQLite storage Every project keeps its own navigation map in ~/.omx/projects/<project-id>/landmark.sqlite.

Extensibility

Markdown files in,
new capabilities out

Workflows, custom agents, skills, and MCP servers are all open-ended. OmniContext CLI is designed to be customized without touching the core.

Custom Workflows

Define your own system prompt and whitelist exactly the tools you want available.

~/.omx/workflows/my-flow.md

Custom Agentic Tools

Write a markdown prompt template plus schema and tool permissions, then call it like any other agent tool.

~/.omx/agents/review.md

Skills

Load reusable prompt instructions as slash commands or through the Skill tool. Claude Code skills work too.

~/.omx/skills/review/SKILL.md

MCP Servers

Connect external tools and data sources over stdio or HTTP, with Claude MCP configs picked up automatically.

~/.omx/mcp.json
