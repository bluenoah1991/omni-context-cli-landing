Tools

Base tools + agent tools

Base tools give the model direct file system and web access. Agent tools run exploratory tasks on a cheaper model -- intermediate tool calls, file contents, and reasoning never enter your main context. You get a concise answer back; the scratch work stays off the books.

Agent Tools

Agent_Explore

Survey project architecture. Maps directory layout, key files, and how the codebase is organized.

Agent_Glance

Preview multiple files at once with brief summaries. Quickly understand what you're working with.

Agent_Search

Search the codebase to answer a specific question. Returns targeted code segments from across the project.

Base Tools

Bash

Native Multi-Protocol

Four API protocols,
zero format conversion

Each LLM API protocol has its own dedicated request builder and stream handler. Prompt caching, extended thinking, structured outputs, and every other provider-specific feature work exactly as the vendor designed them. Anthropic requests are not converted into OpenAI format or vice versa.

Anthropic
Messages API with prompt caching, extended thinking, and streaming. Token-level cache control with custom TTL.
Claude Opus / Sonnet / Haiku
OpenAI
Chat Completions API. Compatible with any endpoint that speaks the OpenAI format -- local models, Zhipu, MiniMax, and more.
GPT-4o / GLM / MiniMax / DeepSeek
Gemini
Native generateContent API with Gemini-specific streaming. No OpenAI shim -- tools and function calling use Gemini's own schema.
Gemini Pro / Flash
Responses API
OpenAI's newer Responses API with built-in tool orchestration. Separate path from Chat Completions, not a compatibility wrapper.
GPT-4o / Codex
Cost Optimization

Context-first design saves real money

Every API call resends your full conversation history. OmniContext CLI attacks cost from four angles -- keeping the context window lean, the rounds few, and the bills low.

Lean system prompts

The built-in system prompts are short, focused, and free of boilerplate. Tool descriptions are minimal. Your context budget goes toward actual work, not framework overhead.

Agentic tools

Exploratory tasks run as agentic tools on a cheaper model. You get a concise answer back; the scratch work stays off the books. The expensive model never does grunt work.

Context editing

Older rounds accumulate tool call payloads and thinking blocks that are no longer relevant. Context editing replaces them with compact placeholders, so the model sees a clean, focused history.

Cross-session memory

Good insights accumulate weight, stale knowledge decays naturally. Accumulated memories are injected automatically so the assistant knows your preferences and project context from the first message.

Cross-Session Memory

It learns as you work

Memory extraction runs automatically when a conversation is compacted. The model reflects on the session, pulls out key points, and evaluates existing ones. Each point carries a score that shifts based on usefulness.

Automatic extraction Key points are pulled from every conversation during compaction and injected into future sessions
Scored memory Helpful points gain +3, neutral ones decay at -1, harmful ones drop fast at -6. Below -10 gets pruned.
Per-project storage Each project has its own memory file at .omx/memory.json. Edit it directly for full control.
memory.json +3
"This project uses TypeScript strict mode with path aliases configured in tsconfig"
memory.json +2
"API routes follow REST conventions in src/routes/ with Zod validation"
memory.json -8
"Uses Webpack for bundling" Decaying -- will be pruned at -10

Extensibility

Everything is a markdown file
or JSON config

Custom workflows, agents, skills, and MCP servers. Build on top of OmniContext CLI without touching the core.

Custom Workflows

Define your own system prompt and control which tools are available. YAML frontmatter + markdown body.

~/.omx/workflows/my-flow.md

Custom Agentic Tools

Write a markdown file with a prompt template, parameter schema, and tool permissions. It becomes a callable agentic tool instantly. Add OMX-AGENTS.md for global agent instructions.

~/.omx/agents/review.md

Skills

Directory-based prompt capabilities invoked as slash commands. Inject instructions for domain-specific knowledge. Compatible with Claude Code skill format.

~/.omx/skills/style/SKILL.md

MCP Servers

Connect external tools and data sources via Model Context Protocol. Stdio and HTTP transports supported. Also reads Claude's MCP config.

~/.omx/mcp.json
